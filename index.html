Overview

ShopSecure is a fictional e-commerce startup with 50 employees, leveraging AWS S3 and EC2. The GRC framework is designed to mitigate risks, achieve ISO 27001/NIST CSF compliance, and reduce incidents by 20% in pre-defined timespan.

AWS Infrastructure Overview

The environment is a single-region AWS setup intentionally configured with minimal security to simulate risks for assessment. It includes:

  • One EC2 instance running Amazon Linux 2, hosting a demo e-commerce app.
  • An S3 bucket with public access enabled for testing exposure risks.
  • No Multi-Factor Authentication (MFA) for IAM users, including the root account.
  • Weak IAM password policy (no complexity or case sensitivity requirements).
  • CloudTrail not enabled, resulting in no API-level activity tracking.
  • Default Security Group with open inbound ports (including SSH) for demonstration.
  • Other default configurations as seen on fresh EC2 deployments.

This is a non-production environment designed solely to identify risks and validate the GRC framework.

Project Workflow

This project followed a structured approach to assess risks, align with frameworks, and build a Governance, Risk, and Compliance (GRC) strategy. The key steps were:

  1. Set up the AWS Infrastructure
    Configured a controlled environment with:
    • EC2 instance running Amazon Linux 2
    • S3 bucket with public access (for testing)
    • No MFA, weak IAM password policy, CloudTrail disabled
    • Default security group rules
  2. Define the Project Scope
    Identified key assets (EC2, S3, IAM) and scoped the evaluation to AWS cloud security posture, identity/access management, and operational monitoring.
  3. Run Security Scans
    Used two tools:
    • Prowler – for AWS CIS benchmark checks and misconfiguration detection.
    • Nessus – for vulnerability scanning of the EC2 instance and network services.
  4. Conduct Risk Analysis and Map to Frameworks
    Findings were mapped to:
    • ISO/IEC 27001:2022 (Information Security Management)
    • NIST Cybersecurity Framework (CSF)
    • COBIT 2019 (Governance and Management Objectives)
    • MITRE ATT&CK (e.g., T1078.004, T1133, T1562.002)
  5. Quantify Risks Using FAIR
    Each critical finding (e.g., Public S3 Buckets, No MFA) was analyzed for financial impact using the FAIR model, resulting in estimated loss ranges.
  6. Design Governance Program
    Developed a governance structure aligning with all frameworks, including:
    • CISO-led oversight and risk tracking
    • KPIs for MFA, S3 access, CloudTrail, and patch management
    • Defined escalation, monitoring, and audit procedures
  7. Communicate Results
    Presented findings and GRC framework through:
    • Formal Presentation for leadership
    • Written Reports (policy, risk analysis, compliance mapping)

GRC Framework Highlights

Governance

CISO oversees AWS security; IT implements mitigations; CEO reviews risks quarterly.

KPIs

  • 100% MFA Adoption
  • 100% S3 Block Public Access
  • 100% CloudTrail Logging
  • 95% Vulnerabilities Patched in 30 Days

Compliance

Aligned with ISO/IEC 27001:2022, COBIT 2019, NIST CSF, and addressing MITRE ATT&CK techniques (T1078.004, T1133, etc.).

Action Plan

  • Enable MFA (€1,000) and S3 Block Public Access (€2,000) by 01/10/2025.
  • Implement multi-region CloudTrail logging (€1,000).
  • Deploy alerting rules (e.g., failed logins >10).
  • Achieve ISO 27001 and NIST CSF certification within 6 months.
  • More details plans can be found in the attachments.
